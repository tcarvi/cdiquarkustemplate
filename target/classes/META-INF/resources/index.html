Design deste projeto

Padrão Resources vs Services

Classes que executam processos administrativos de inicialização ou de configurações, possuem o sufixo Resource.

Já as classes que apresentam dados solicitados pelo processamento da API possuem o sufixo Service.

Normalmente informações mais negociais encontram-se nas classes de padrão Service. Estas classes são injetadas no resource.

Desenvolvedor deve pensar no fluxo de vida dos beans e indicar o escopo de sua permanência na memória RAM. Singleton, ApplicationScoped, SessionScoped, RequestScoped or DependentScoped.
